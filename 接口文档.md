# 简介

本项目是「证据链股权结构抽取与展示」后端服务，提供文件上传、AI 抽取生成股权结构图、来源查询与文本高亮等接口。  
所有接口统一使用 `ApiResponse<T>` 返回：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... 实际业务数据 ... }
}
```

---

# 接口总览

| 序号 | 接口名称       | 方法 | URL                   | 说明                                 |
| ---- | -------------- | ---- | --------------------- | ------------------------------------ |
| 1    | 上传文件       | POST | `/api/files/upload`   | 上传一个或多个文件，返回 fileId 列表及文件信息 |
| 2    | 生成股权结构图 | POST | `/api/graph/generate` | 根据 fileIds 调用 AI 生成股权结构图 |
| 3    | 获取来源列表   | GET  | `/api/source`         | 根据实体/关系/冲突 ID 获取来源文件及片段 |
| 4    | 文本高亮       | POST | `/api/highlight`      | 根据 fileId 与片段返回全文及高亮范围 |

# 接口具体参数与示例

## 1. 上传文件 - POST `/api/files/upload`

- 请求方式：`POST`  
- Content-Type：`multipart/form-data`  
- 说明：支持多文件上传，返回每个文件的 `fileId` 和基础信息。

### 请求参数

| 参数名 | 位置              | 类型              | 必填 | 说明                           |
| ------ | ----------------- | ----------------- | ---- | ------------------------------ |
| files  | form-data（表单） | `MultipartFile[]` | 是   | 上传的文件数组，字段名固定为 `files` |

### 具体输入示例

multipart/form-data：

```text
files: doc1.pdf
files: doc2.pdf
```

### 具体输出示例（成功）

业务数据为 `FileUploadResponse`：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "fileIds": [1, 2],
    "files": [
      {
        "fileId": 1,
        "fileName": "file1.txt",
        "size": 2345
      },
      {
        "fileId": 2,
        "fileName": "file2.txt",
        "size": 1567
      }
    ]
  }
}
```

## 2. 生成股权结构图 - POST `/api/graph/generate`

- 请求方式：`POST`  
- Content-Type：`application/json`  
- 说明：根据上传得到的 `fileIds` 触发 AI 抽取，生成股权结构图，并将结果写入数据库，返回图谱结构数据。

### 请求参数（Body）

`GraphGenerateRequest`：

| 字段名  | 类型     | 必填 | 说明          |
| ------- | -------- | ---- | ------------- |
| fileIds | `Long[]` | 是   | 文件 ID 列表，不能为空 |

### 具体输入示例

```json
{
  "fileIds": [1, 2]
}
```

## 3. 获取来源列表 - GET `/api/source`

- 请求方式：`GET`  
- 说明：根据目标类型和目标 ID 查询对应的来源列表，用于前端展示可选来源（点击后再去高亮）。

### 请求参数（Query）

| 参数名     | 类型     | 必填 | 说明                                      |
| ---------- | -------- | ---- | ----------------------------------------- |
| targetType | `String` | 是   | 目标类型：`ENTITY` / `RELATION` / `CONFLICT` |
| targetId   | `Long`   | 是   | 目标 ID（实体 ID / 关系 ID / 冲突 ID）   |

### 具体输入示例（ENTITY 实体来源）

```http
GET /api/source?targetType=ENTITY&targetId=1
```

### 具体输出示例（ENTITY 实体来源）

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "data": [
      {
        "fileId": 1,
        "fileName": "gsxx_2023.pdf",
        "snippet": "公司A为有限责任公司，注册资本人民币 1000 万元。"
      }
    ]
  }
}
```

---

### 具体输入示例（RELATION 关系来源）

### 具体输出示例（RELATION 关系来源）

## 4. 文本高亮 - POST `/api/highlight`

- 请求方式：`POST`  
- Content-Type：`application/json`  
- 说明：前端选择一个来源后，传入 `fileId` 和 `snippet`，后端返回全文内容。

### 请求参数（Body）

`HighlightRequest`：

| 字段名 | 类型     | 必填 | 说明                |
| ------ | -------- | ---- | ------------------- |
| fileId | `Long`   | 是   | 文件 ID             |
| snippet| `String` | 是   | 需要高亮的文本片段  |

### 具体输入示例

```json
{
  "fileId": 1,![1773065415747](image/接口文档/1773065415747.png)![1773065419001](image/接口文档/1773065419001.png)![1773065454638](image/接口文档/1773065454638.png)
  "snippet": "公司A持有公司B 60%的股份"
}
```

### 具体输出示例（成功）

业务数据为 `HighlightResponse`：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "fileId": 1,
    "fileName": "file1.docx",
    "content": "公司A持有公司B 60%的股份。公司B为公司C的全资子公司。"
  }
}
```

# 错误返回

所有接口在发生错误时，统一返回结构仍为 `ApiResponse`：

```json
{
  "code": 1001,
  "message": "参数错误",
  "data": null
}
```

## 通用错误码列表（`ErrorCode` 枚举）

- `SUCCESS (0, "success")`：成功  
- `PARAM_ERROR (1001, "参数错误")`：常见于参数缺失/格式错误，如 `fileIds` 为空、`fileId`/`snippet`/`targetType`/`targetId` 为空等  
- `FILE_NOT_FOUND (2001, "文件不存在")`：根据 `fileId` 未找到对应文件记录  
- `GRAPH_NOT_FOUND (2002, "图谱不存在")`：根据图谱 ID 未找到记录（如后续扩展图谱查询接口时）  
- `AI_GENERATION_FAILED (3001, "AI 抽取失败")`：调用大模型失败或返回结果无法解析（多见于 `/api/graph/generate`）  
- `SYSTEM_ERROR (5000, "系统异常")`：未捕获异常、数据库故障等内部错误
